# Task Manager — CW-03

Flutter task management app backed by Firebase Firestore with full CRUD and nested subtasks.

## Setup

1. **Create your Firebase project** at [console.firebase.google.com](https://console.firebase.google.com).
2. **Configure FlutterFire:**
    ```bash
    dart pub global activate flutterfire_cli
    flutterfire configure
    ```
    This overwrites `lib/firebase_options.dart` with your real credentials.
3. **Enable Firestore** in the Firebase Console (test mode for development).
4. **Install dependencies:**
    ```bash
    flutter pub get
    ```
5. **Run:**
    ```bash
    flutter run
    ```

## Enhanced Features

### 1. Real-Time Search / Filter

A search bar above the task list filters tasks by title as you type — no page refresh needed. The filter runs client-side against the live Firestore stream, so it works instantly even as tasks are added or removed.

### 2. Dark Mode (ThemeMode.system)

The app follows the device's dark/light preference automatically via `ThemeMode.system` in `MaterialApp`. No toggle needed — it updates the moment the user changes their system theme.

## Folder Structure

```
lib/
  models/       task.dart           — Task data model (toMap, fromMap, copyWith)
  services/     task_service.dart   — All Firestore CRUD operations
  screens/      task_list_screen.dart — Main UI with StreamBuilder
  widgets/      task_tile.dart      — Individual task row with subtasks
  main.dart     — Firebase init + MaterialApp
  firebase_options.dart — Generated by flutterfire configure
```

## Known Limitations

- No authentication — all users share the same Firestore collection (acceptable for a class project in test mode; tighten rules before any real deployment).
- No pagination — large task lists will load all documents at once.
- Subtasks are stored as a nested array inside the parent task document, so very large subtask lists could approach Firestore's 1 MB document limit.
